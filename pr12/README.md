# Практические работы

## Практика 7

В этой работе было создано базовое серверное приложение на Node.js с использованием Express. Приложение реализует REST API для двух сущностей — пользователей и товаров. Пользователь может зарегистрироваться, указав email, имя, фамилию и пароль (пароль хешируется через bcrypt), а затем войти в систему. Для товаров реализован полный набор CRUD-операций: создание, получение списка, получение по id, обновление и удаление. Данные хранятся в оперативной памяти.

## Практика 8

Здесь серверное приложение было доработано — добавлена авторизация через JWT-токены. Теперь при входе в систему пользователь получает токен, который необходимо передавать в заголовке Authorization для доступа к защищённым маршрутам. Был добавлен маршрут GET /api/auth/me, возвращающий информацию о текущем авторизованном пользователе. Защита также распространяется на получение товара по id, обновление и удаление товара — без токена эти операции недоступны.

## Практика 9

В этой практике реализован механизм refresh-токенов. Теперь при логине сервер возвращает пару токенов — accessToken (короткоживущий, 15 минут) и refreshToken (7 дней). Добавлен маршрут POST /api/auth/refresh, который принимает refresh-токен из заголовка и генерирует новую пару. При этом старый refresh-токен инвалидируется (хранится в Set на сервере), что не позволяет использовать его повторно.

## Практика 10

К серверной части было добавлено полноценное клиентское приложение на React (сборка через Vite). Фронтенд включает страницы входа и регистрации, а после авторизации пользователь попадает на страницу управления товарами. Там можно просматривать список товаров, создавать новые, открывать детальную информацию по конкретному товару, редактировать и удалять. Для работы с API используется axios с interceptors — при получении 401 ошибки клиент автоматически пытается обновить токены через refresh-эндпоинт. Vite проксирует запросы к /api на Express-сервер.

## Практика 11

Финальная доработка — добавлена система ролей с тремя уровнями доступа: пользователь (user), продавец (seller) и администратор (admin). Роль выбирается при регистрации. Обычный пользователь может только просматривать товары. Продавец, помимо просмотра, может создавать и редактировать товары. Администратор имеет полный доступ, включая удаление товаров, а также может управлять пользователями — просматривать список, менять им роли и блокировать. На фронтенде кнопки действий отображаются в зависимости от роли текущего пользователя, а администратору доступна отдельная страница управления пользователями.
