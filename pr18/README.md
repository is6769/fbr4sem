# Практики 13–17: Прогрессивное веб-приложение «Список задач»

## Общее описание

Серия практических работ посвящена поэтапному созданию Progressive Web Application (PWA) — приложения для управления списком задач. Каждая практика расширяет функциональность предыдущей, демонстрируя ключевые технологии PWA.

---

## Практика 13 — Базовое PWA

**Цель:** создание простого офлайн-приложения со списком задач.

**Что реализовано:**
- HTML/CSS/JS-приложение для создания, отметки и удаления задач
- Хранение данных в `localStorage`
- Регистрация Service Worker с кэширующей стратегией cache-first
- Полная офлайн-поддержка: приложение работает без сети

**Файлы:** `index.html`, `app.js`, `styles.css`, `sw.js`

---

## Практика 14 — Установка PWA (Web App Manifest и иконки)

**Цель:** сделать приложение устанавливаемым на устройство.

**Что добавлено поверх практики 13:**
- Файл `manifest.json` с метаданными приложения (имя, цвета, иконки, режим отображения)
- Скрипт `generate-icons.js` для программной генерации иконок (72x72, 192x192, 512x512)
- Приложение можно «установить» на рабочий стол через браузер

**Новые файлы:** `manifest.json`, `generate-icons.js`, `icons/`

---

## Практика 15 — App Shell и HTTPS-сервер

**Цель:** внедрить архитектуру App Shell и серверную раздачу по HTTPS.

**Что добавлено поверх практики 14:**
- HTTPS-сервер на Express.js с SSL-сертификатами
- Паттерн App Shell в Service Worker: core-ресурсы кэшируются при установке, контентные страницы загружаются network-first
- Вторая страница `about.html` с навигацией между страницами
- Улучшенный интерфейс с вкладками в шапке

**Новые файлы:** `server.js`, `package.json`, `about.html`, `cert.pem`, `key.pem`

---

## Практика 16 — Socket.IO и Push-уведомления

**Цель:** добавить синхронизацию в реальном времени и push-уведомления.

**Что добавлено поверх практики 15:**
- Интеграция Socket.IO: при добавлении задачи одним пользователем она мгновенно появляется у всех подключённых клиентов
- Web Push Notifications через библиотеку `web-push` с VAPID-ключами
- Серверные эндпоинты `/subscribe` и `/unsubscribe` для управления push-подписками
- Панель управления push-уведомлениями в интерфейсе (подписка/отписка)
- Service Worker обрабатывает входящие push-сообщения и клик по уведомлению

**Новые зависимости:** `socket.io`, `web-push`, `dotenv`

---

## Практика 17 — Напоминания

**Цель:** добавить функционал напоминаний с push-уведомлениями по расписанию.

**Что добавлено поверх практики 16:**
- Форма для создания задачи с напоминанием (текст + дата/время через `datetime-local`)
- Расширенная структура данных в `localStorage`: каждая заметка содержит уникальный `id` и поле `reminder` (timestamp)
- Серверное планирование push-уведомлений через `setTimeout` с хранением активных таймеров в `Map`
- API-эндпоинты: `/api/reminders/schedule`, `/api/reminders/snooze`, `DELETE /api/reminders/:taskId`
- Обработка действий уведомлений в Service Worker: кнопка «Отложить на 5 минут» отправляет запрос `/api/reminders/snooze` и перепланирует таймер
- Напоминания работают даже при закрытом приложении — таймеры живут на сервере и отправляют push через web-push
- При открытии приложения все будущие напоминания автоматически перепланируются на сервере
